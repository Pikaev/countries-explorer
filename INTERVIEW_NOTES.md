# Шпаргалка для собеседований

## День 1: Event Loop и асинхронность

**Вопрос:** Как JS выполняет асинхронный код и почему `await` не блокирует поток?

**Ответ своими словами:**

JS однопоточный. Event Loop перекидывает задачи из очереди в стек, когда стек пуст. 

`await` не блокирует главный поток. Он приостанавливает выполнение текущей функции и возвращает управление Event Loop'у, позволяя обрабатывать другие задачи, пока промис не резолвится.

**Пример из проекта:**

В `countries-explorer` при фетчинге списка стран на сервере (Next.js) мы используем `await fetch()`. Пока идёт сеть, сервер не зависает, а Event Loop может обрабатывать другие входящие запросы. Если бы мы использовали синхронный blocking-код, весь сервер лёг бы. **(читать)**

## День 2: React Reconciliation и Virtual DOM
**Вопрос:** Как React обновляет DOM и почему `key={index}` в списках — это антипаттерн?

**Ответ своими словами:**
React не перерисовывает весь DOM. Он строит Virtual DOM, сравнивает его с предыдущим (diffing) и точечно патчит реальный DOM. Это экономит ресурсы браузера.
`key` нужен, чтобы React понимал идентичность элементов. Если использовать `index`, при фильтрации/сортировке React перепутает элементы (подумает, что они мутировали, а не сдвинулись). Это ломает локальный стейт и убивает производительность. Всегда используй стабильный ID из данных.

**Пример из проекта:**
В `countries-explorer` при рендере списка стран используем `key={country.cca3}` (уникальный ISO-код), а не индекс массива. Список сортируется по населению, и стабильный ключ гарантирует, что React корректно сопоставит карточки при перерисовке.
