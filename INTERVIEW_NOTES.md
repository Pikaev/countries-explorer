# Шпаргалка для собеседований

## День 1: Event Loop и асинхронность

**Вопрос:** Как JS выполняет асинхронный код и почему `await` не блокирует поток?

**Ответ своими словами:**

JS однопоточный. Event Loop перекидывает задачи из очереди в стек, когда стек пуст. 

`await` не блокирует главный поток. Он приостанавливает выполнение текущей функции и возвращает управление Event Loop'у, позволяя обрабатывать другие задачи, пока промис не резолвится.

**Пример из проекта:**

В `countries-explorer` при фетчинге списка стран на сервере (Next.js) мы используем `await fetch()`. Пока идёт сеть, сервер не зависает, а Event Loop может обрабатывать другие входящие запросы. Если бы мы использовали синхронный blocking-код, весь сервер лёг бы. **(читать)**
